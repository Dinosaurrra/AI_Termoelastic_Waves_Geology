\section{Conditional MeshGraphNet for Thermoelastic Wave Prediction}

\subsection{Role of MeshGraphNet in the AI Pipeline}

The Conditional MeshGraphNet model is used in this work as a graph-based supervised surrogate model for thermoelastic wave prediction in geological media. Its purpose is not to replace the physical formulation of the thermoelastic problem, but to learn a fast approximation of the state-transition operator generated by COMSOL simulations.

In the overall AI pipeline, COMSOL provides reference finite-element trajectories of temperature, displacement, velocity, stress, and strain fields. These trajectories are then converted into graph-based training samples. The neural network receives the current state of the simulated medium and predicts the next state or the increment of the state. Therefore, the model learns the data-driven transition

\[
\hat{\mathbf{Y}}^{t+\Delta t}
=
f_{\theta}
\left(
\mathbf{X}^{t},
\mathcal{G}
\right),
\]

where \( \mathbf{X}^{t} \) is the node feature matrix at time \( t \), \( \mathcal{G} \) is the graph representation of the COMSOL mesh, and \( \hat{\mathbf{Y}}^{t+\Delta t} \) is the predicted matrix of physical fields at the next time step.

The model is not a physics-informed neural network in the strict sense. The training objective does not contain an explicit residual of the governing thermoelastic partial differential equations. Instead, the model is trained in a supervised way by comparing its predictions with COMSOL-generated targets. The connection with physics is introduced through the finite-element mesh structure, node and edge features, material and scenario conditioning, and local message passing between neighboring mesh nodes.

\subsection{Graph Representation of the COMSOL Mesh}

The COMSOL finite-element mesh is represented as a graph

\[
\mathcal{G} = (\mathcal{V}, \mathcal{E}),
\]

where \( \mathcal{V} \) is the set of graph nodes and \( \mathcal{E} \) is the set of graph edges. Each node corresponds to a spatial point of the finite-element discretization, while each edge represents a local connection between neighboring nodes in the mesh.

For a mesh with \( N \) nodes, the node set is written as

\[
\mathcal{V}
=
\left\{
v_i
\right\}_{i=1}^{N}.
\]

The edge set contains directed or undirected neighborhood relations between nodes:

\[
\mathcal{E}
=
\left\{
(i,j)
\mid
v_i, v_j \in \mathcal{V}
\right\}.
\]

In the implementation, graph connectivity is stored in \( \texttt{edge\_index} \). The first row contains source-node indices, and the second row contains destination-node indices. Thus, for an edge \( (i,j) \), information is sent from node \( i \) to node \( j \).

Each node has a spatial coordinate vector

\[
\mathbf{p}_i
=
\begin{bmatrix}
x_i \\
y_i \\
z_i
\end{bmatrix}.
\]

For each edge \( (i,j) \), the relative position vector is defined as

\[
\mathbf{r}_{ij} = \mathbf{p}_j - \mathbf{p}_i.
\]

The corresponding edge length is

\[
d_{ij} = \|\mathbf{r}_{ij}\|_2.
\]

The edge feature vector combines relative direction and distance:

\[
\mathbf{a}_{ij}
=
\begin{bmatrix}
\mathbf{r}_{ij} \\
d_{ij}
\end{bmatrix}.
\]

This representation preserves the local geometry of the finite-element mesh. It allows the model to learn how information propagates between neighboring spatial locations without converting the mesh into a regular Cartesian grid. This is important because thermoelastic wave propagation is local: temperature variation, deformation, stress redistribution, and wave motion are all influenced by the state of nearby regions.

\subsection{Node and Edge Features}

The input feature vector of node \( i \) at time \( t \) combines geometry, material information, scenario parameters, and the current dynamic state:

\[
\mathbf{x}_i^t
=
\begin{bmatrix}
\mathbf{p}_i \\
\mathbf{m}_i \\
\mathbf{s} \\
\mathbf{q}_i^t
\end{bmatrix}.
\]

Here, \( \mathbf{p}_i \) denotes the spatial coordinates of the node, \( \mathbf{m}_i \) denotes material parameters, \( \mathbf{s} \) denotes scenario-level conditioning variables, and \( \mathbf{q}_i^t \) denotes the current physical state at the node.

The material vector may contain parameters such as thermal conductivity, density, heat capacity, Young's modulus, Poisson's ratio, and thermal expansion coefficient:

\[
\mathbf{m}_i
=
\begin{bmatrix}
k_i \\
\rho_i \\
c_{p,i} \\
E_i \\
\nu_i \\
\alpha_i
\end{bmatrix}.
\]

The scenario vector \( \mathbf{s} \) describes global simulation conditions, for example the source temperature, source location, heating radius, loading configuration, or other parameters that define a particular experiment. In the implemented pipeline, this conditional information is already concatenated into \( \mathbf{x}_i^t \). Therefore, the model does not receive a separate condition branch; it processes a unified node-feature vector.

The dynamic state vector may include temperature, displacement, velocity, stress, and strain components:

\[
\mathbf{q}_i^t
=
\begin{bmatrix}
T_i^t \\
u_i^t \\
v_i^t \\
w_i^t \\
\dot{u}_i^t \\
\dot{v}_i^t \\
\dot{w}_i^t \\
\sigma_{x,i}^t \\
\sigma_{y,i}^t \\
\sigma_{z,i}^t \\
\sigma_{xy,i}^t \\
\sigma_{yz,i}^t \\
\sigma_{xz,i}^t \\
\varepsilon_{x,i}^t \\
\varepsilon_{y,i}^t \\
\varepsilon_{z,i}^t
\end{bmatrix}.
\]

The complete node feature matrix is

\[
\mathbf{X}^{t}
=
\begin{bmatrix}
(\mathbf{x}_1^t)^{\top} \\
(\mathbf{x}_2^t)^{\top} \\
\vdots \\
(\mathbf{x}_N^t)^{\top}
\end{bmatrix}
\in
\mathbb{R}^{N \times F_v},
\]

where \( F_v \) is the number of node features.

The edge feature matrix is

\[
\mathbf{A}
=
\begin{bmatrix}
\mathbf{a}_{1}^{\top} \\
\mathbf{a}_{2}^{\top} \\
\vdots \\
\mathbf{a}_{M}^{\top}
\end{bmatrix}
\in
\mathbb{R}^{M \times F_e},
\]

where \( M \) is the number of graph edges and \( F_e \) is the number of edge features. In the implemented data representation, edge attributes contain geometric information derived from relative coordinates and distances.

\subsection{Prediction Target and Delta Formulation}

The model approximates the COMSOL state-transition operator. For the full graph, the supervised prediction problem is written as

\[
\hat{\mathbf{Y}}^{t+\Delta t}
=
f_{\theta}
\left(
\mathbf{X}^{t},
\mathcal{G}
\right).
\]

The output has the form

\[
\hat{\mathbf{Y}}^{t+\Delta t}
\in
\mathbb{R}^{N \times C},
\]

where \( C \) is the number of predicted physical fields.

The implementation supports a delta formulation. In this case, the neural network predicts the increment of the dynamic state rather than the absolute future state:

\[
\Delta \hat{\mathbf{q}}_i^t
=
f_{\theta}
\left(
\mathbf{x}_i^t,
\mathcal{G}
\right).
\]

The next state is reconstructed as

\[
\hat{\mathbf{q}}_i^{t+\Delta t}
=
\mathbf{q}_i^t
+
\Delta \hat{\mathbf{q}}_i^t.
\]

This formulation is suitable for time-dependent thermoelastic trajectories because changes between adjacent time steps are often smoother than the full physical fields. During autoregressive inference, the predicted next state is inserted back into the node features and used as input for the following step.

For a rollout of \( R \) prediction steps, the recursive update can be written as

\[
\hat{\mathbf{q}}^{t+(r+1)\Delta t}
=
\hat{\mathbf{q}}^{t+r\Delta t}
+
f_{\theta}
\left(
\hat{\mathbf{X}}^{t+r\Delta t},
\mathcal{G}
\right),
\qquad
r = 0,1,\dots,R-1.
\]

Here, \( \hat{\mathbf{X}}^{t+r\Delta t} \) contains static features and the predicted dynamic state from the previous rollout step.

\subsection{Encoder--Processor--Decoder Architecture}

The implemented model follows the Encoder--Processor--Decoder architecture:

\[
f_{\theta}
=
D_{\theta_D}
\circ
P_{\theta_P}
\circ
E_{\theta_E}.
\]

The encoder \( E_{\theta_E} \) maps raw node and edge features into a latent space. The processor \( P_{\theta_P} \) applies several message passing blocks over the graph. The decoder \( D_{\theta_D} \) maps the final node latent vectors into predicted physical fields.

The initial node and edge embeddings are computed as

\[
\mathbf{h}_i^0
=
\phi_v^{enc}
\left(
\mathbf{x}_i^t
\right),
\]

\[
\mathbf{e}_{ij}^0
=
\phi_e^{enc}
\left(
\mathbf{a}_{ij}
\right).
\]

Here, \( \mathbf{h}_i^0 \) is the latent representation of node \( i \), and \( \mathbf{e}_{ij}^0 \) is the latent representation of edge \( (i,j) \).

The processor consists of \( K \) message passing blocks:

\[
P_{\theta_P}
=
P_{\theta_P}^{K-1}
\circ
\dots
\circ
P_{\theta_P}^{1}
\circ
P_{\theta_P}^{0}.
\]

After the final message passing step, the decoder produces the output for each node:

\[
\hat{\mathbf{y}}_i
=
\phi^{dec}
\left(
\mathbf{h}_i^K
\right).
\]

The model output for all nodes is

\[
\hat{\mathbf{Y}}
=
\begin{bmatrix}
\hat{\mathbf{y}}_1^{\top} \\
\hat{\mathbf{y}}_2^{\top} \\
\vdots \\
\hat{\mathbf{y}}_N^{\top}
\end{bmatrix}.
\]

The multilayer perceptrons used in the model consist of linear layers, optional layer normalization, SiLU activation, and dropout. A compact representation of one hidden transformation is

\[
\mathbf{z}_{\ell+1}
=
\operatorname{Dropout}
\left(
\operatorname{SiLU}
\left(
\operatorname{LayerNorm}
\left(
\mathbf{W}_{\ell}\mathbf{z}_{\ell}
+
\mathbf{b}_{\ell}
\right)
\right)
\right).
\]

The SiLU activation is defined as

\[
\operatorname{SiLU}(x)
=
x \cdot \sigma(x).
\]

This architecture is appropriate for continuous physical-field regression because it combines nonlinear feature transformation with stable latent updates.

\subsection{Message Passing Mechanism}

The processor is the central part of Conditional MeshGraphNet. Each message passing block updates edge states, aggregates messages over destination nodes, and then updates node states.

At message passing step \( k \), the source and destination nodes are obtained from \( \texttt{edge\_index} \). For an edge \( (i,j) \), the edge update uses the source node latent vector, the destination node latent vector, and the current edge latent vector:

\[
\tilde{\mathbf{e}}_{ij}^{k+1}
=
\phi_e^k
\left(
\left[
\mathbf{h}_i^k,
\mathbf{h}_j^k,
\mathbf{e}_{ij}^k
\right]
\right).
\]

A residual connection and layer normalization are then applied:

\[
\mathbf{e}_{ij}^{k+1}
=
\operatorname{LayerNorm}
\left(
\mathbf{e}_{ij}^{k}
+
\tilde{\mathbf{e}}_{ij}^{k+1}
\right).
\]

The updated edge representation can be interpreted as a learned local interaction between two neighboring mesh nodes. It encodes how information should be transferred from the source node to the destination node using both node states and edge geometry.

Messages are aggregated over destination nodes using a summation operation:

\[
\mathbf{m}_j^{k+1}
=
\sum_{i:(i,j)\in\mathcal{E}}
\mathbf{e}_{ij}^{k+1}.
\]

In the implementation, this operation corresponds to scatter-add over the destination indices. It collects all incoming edge messages for each node.

The node update then combines the current node latent vector with the aggregated message:

\[
\tilde{\mathbf{h}}_j^{k+1}
=
\phi_v^k
\left(
\left[
\mathbf{h}_j^k,
\mathbf{m}_j^{k+1}
\right]
\right).
\]

The final node representation is obtained using a residual connection and layer normalization:

\[
\mathbf{h}_j^{k+1}
=
\operatorname{LayerNorm}
\left(
\mathbf{h}_j^k
+
\tilde{\mathbf{h}}_j^{k+1}
\right).
\]

After \( K \) message passing steps, the latent vector of each node contains information from a \( K \)-hop neighborhood:

\[
\mathcal{N}_K(i)
=
\left\{
j \in \mathcal{V}
\mid
\operatorname{dist}_{\mathcal{G}}(i,j) \leq K
\right\}.
\]

This is physically meaningful because thermoelastic effects propagate through local interactions. The model does not explicitly solve the thermoelastic equations, but the graph topology and message passing mechanism allow it to learn local transition patterns from COMSOL trajectories.

\subsection{Loss Function and Training Procedure}

The model is trained as a supervised regression model. The prediction is compared with the COMSOL target using a weighted mean squared error over physical fields:

\[
\mathcal{L}
=
\frac{1}{\sum_{c=1}^{C} w_c}
\sum_{c=1}^{C}
w_c
\cdot
\frac{1}{N}
\sum_{i=1}^{N}
\left(
\hat{y}_{i,c}
-
y_{i,c}
\right)^2.
\]

Here, \( \hat{y}_{i,c} \) is the predicted value of field \( c \) at node \( i \), \( y_{i,c} \) is the COMSOL target value, and \( w_c \) is the weight assigned to the field.

The implementation groups output fields into physical categories:

\[
\mathcal{C}
=
\mathcal{C}_{T}
\cup
\mathcal{C}_{u}
\cup
\mathcal{C}_{\dot{u}}
\cup
\mathcal{C}_{\sigma}
\cup
\mathcal{C}_{\varepsilon}
\cup
\mathcal{C}_{other}.
\]

These groups correspond to temperature, displacement, velocity, stress, strain, and other fields. A field-specific weight is selected according to the group:

\[
w_c = w_g,
\qquad
c \in \mathcal{C}_g.
\]

This is useful because different physical quantities have different numerical ranges and different importance for the analysis. For example, stress fields may be emphasized when the goal is to detect mechanically critical regions, while temperature fields may be emphasized when the goal is to track thermal diffusion.

Training uses the Adam optimizer. At each optimization step, the model receives a sample containing node features, target fields, graph connectivity, and edge attributes:

\[
\left(
\mathbf{X}^{t},
\mathbf{Y}^{t+\Delta t},
\texttt{edge\_index},
\texttt{edge\_attr}
\right).
\]

The training loss for one epoch is averaged over the training samples:

\[
\mathcal{L}_{train}^{(e)}
=
\frac{1}{|\mathcal{D}_{train}|}
\sum_{s \in \mathcal{D}_{train}}
\mathcal{L}^{(s)}.
\]

The validation loss is computed without parameter updates:

\[
\mathcal{L}_{val}^{(e)}
=
\frac{1}{|\mathcal{D}_{val}|}
\sum_{s \in \mathcal{D}_{val}}
\mathcal{L}^{(s)}.
\]

The best model is selected according to the lowest validation loss:

\[
e^{*}
=
\arg\min_{e}
\mathcal{L}_{val}^{(e)}.
\]

The training procedure also includes gradient clipping:

\[
\mathbf{g}
\leftarrow
\mathbf{g}
\cdot
\min
\left(
1,
\frac{\tau}{\|\mathbf{g}\|_2}
\right),
\]

where \( \tau \) is the maximum allowed gradient norm. This stabilizes training when large gradients appear.

When CUDA is available, automatic mixed precision is used to reduce memory usage and improve computational efficiency. Early stopping is applied when the validation loss does not improve for a predefined number of epochs. Checkpoints store the model parameters, optimizer state, validation loss, field names, input-output dimensions, dataset identifiers, model configuration, and target mode.

The loss function is purely supervised. It does not contain an explicit PDE residual term. Therefore, the model learns to approximate COMSOL-generated transitions rather than directly enforcing the governing thermoelastic equations during optimization.

\subsection{Fine-Tuning Strategy}

Fine-tuning is supported to adapt the model to new datasets, rock types, temperature ranges, or scenario configurations. The model parameters can be divided into encoder, processor, and decoder parameters:

\[
\theta
=
\left\{
\theta_E,
\theta_P,
\theta_D
\right\}.
\]

The implementation supports three fine-tuning modes.

In full fine-tuning, all parameters remain trainable:

\[
\theta_{train}
=
\left\{
\theta_E,
\theta_P,
\theta_D
\right\}.
\]

In decoder-only fine-tuning, the encoder and processor are frozen, while only the decoder is updated:

\[
\theta_{train}
=
\left\{
\theta_D
\right\}.
\]

This mode is useful when the latent representation and message passing dynamics remain valid, but the final mapping to output fields needs adaptation.

In processor-decoder fine-tuning, the encoder is frozen, while the processor and decoder are updated:

\[
\theta_{train}
=
\left\{
\theta_P,
\theta_D
\right\}.
\]

This mode is useful when the input feature structure remains consistent, but the local propagation dynamics or material response changes. Since the processor is responsible for message passing, updating it allows the model to adapt to new interaction patterns without retraining the entire architecture from scratch.

\subsection{Validation and Physical Evaluation}

The validation procedure includes both statistical and physical evaluation. The first level is one-step validation, where the model predicts the next state from the true current state:

\[
\mathbf{X}^{t}
\rightarrow
\hat{\mathbf{Y}}^{t+\Delta t}.
\]

This measures the local transition accuracy of the model.

The second level is autoregressive rollout validation. In this case, the model repeatedly uses its own previous prediction as input:

\[
\hat{\mathbf{q}}^{t+(r+1)\Delta t}
=
\hat{\mathbf{q}}^{t+r\Delta t}
+
\Delta \hat{\mathbf{q}}^{t+r\Delta t}.
\]

This evaluation is more demanding because prediction errors can accumulate over time. A model may perform well in one-step prediction but become unstable during long rollouts.

The statistical metrics include mean squared error, root mean squared error, mean absolute error, and per-field errors:

\[
\mathrm{RMSE}_c
=
\sqrt{
\frac{1}{N}
\sum_{i=1}^{N}
\left(
\hat{y}_{i,c}
-
y_{i,c}
\right)^2
},
\]

\[
\mathrm{MAE}_c
=
\frac{1}{N}
\sum_{i=1}^{N}
\left|
\hat{y}_{i,c}
-
y_{i,c}
\right|.
\]

In addition to raw-field metrics, derived physical quantities are computed. These include displacement magnitude,

\[
\|\mathbf{u}_i\|_2
=
\sqrt{
(u_i)^2
+
(v_i)^2
+
(w_i)^2
},
\]

velocity magnitude,

\[
\|\dot{\mathbf{u}}_i\|_2
=
\sqrt{
(\dot{u}_i)^2
+
(\dot{v}_i)^2
+
(\dot{w}_i)^2
},
\]

and temperature change,

\[
\Delta T_i^t
=
T_i^t
-
T_i^{0}.
\]

The validation pipeline also evaluates von Mises stress. If the COMSOL field \( \sigma_{\mathrm{vM}} \) is exported directly, it is used as a derived quantity. Otherwise, it can be reconstructed from stress components:

\[
\sigma_{\mathrm{vM}}
=
\sqrt{
\frac{1}{2}
\left[
(\sigma_x-\sigma_y)^2
+
(\sigma_y-\sigma_z)^2
+
(\sigma_z-\sigma_x)^2
+
6
\left(
\sigma_{xy}^2
+
\sigma_{xz}^2
+
\sigma_{yz}^2
\right)
\right]
}.
\]

Risk zones are estimated from normalized stress intensity. For example, an \( 80\% \) stress-risk flag can be written as

\[
R_i^t
=
\mathbb{I}
\left(
\frac{\sigma_{\mathrm{vM},i}^{t}}
{\max_j \sigma_{\mathrm{vM},j}^{t}}
\geq
0.8
\right).
\]

The validation report also includes error-over-time tables and visual comparison slices. A typical comparison figure contains the COMSOL reference field, the MeshGraphNet prediction, and the absolute error:

\[
\left|
\hat{y}_{i,c}^{t}
-
y_{i,c}^{t}
\right|.
\]

For wave propagation analysis, the wave-front radius can be estimated from a thresholded temperature-change response. Let \( \mathbf{p}_{src} \) be the source center and let \( \mathcal{A}^{t} \) be the set of active nodes at time \( t \). Then the front radius can be approximated as

\[
R_{front}^{t}
=
\operatorname{Percentile}_{95}
\left(
\left\{
\|\mathbf{p}_i-\mathbf{p}_{src}\|_2
\mid
i \in \mathcal{A}^{t}
\right\}
\right).
\]

This physical validation is important because global error metrics alone may hide local inaccuracies in wave-front motion, stress concentration, or high-risk regions.

\subsection{Advantages and Limitations}

The main advantage of Conditional MeshGraphNet is its ability to process unstructured COMSOL meshes directly. Unlike grid-based models, it does not require interpolation of the finite-element mesh into a regular image-like domain. The graph representation preserves local connectivity, which is essential for learning propagation patterns in a discretized physical medium.

The second advantage is material and scenario conditioning. Since coordinates, material properties, scenario parameters, and current dynamic fields are included in the node features, the model can learn different responses for different physical conditions while keeping the same general architecture.

The third advantage is local message passing. Thermoelastic effects propagate through neighboring regions of the medium, and message passing provides a natural mechanism for learning such local interactions from simulation data.

The fourth advantage is efficient inference. After training, the model predicts the next state by a forward pass through the neural network, which is typically faster than running a full COMSOL simulation for each new time step or scenario.

However, the approach also has limitations. First, it depends on the quality and diversity of the COMSOL dataset. If a new scenario lies far outside the training distribution, the model may extrapolate poorly:

\[
\left(
\mathbf{X}_{new},
\mathcal{G}_{new}
\right)
\not\sim
\mathcal{D}_{train}.
\]

Second, the model does not strictly enforce conservation laws, constitutive equations, or boundary conditions during inference. These constraints are learned implicitly from data rather than imposed explicitly through the loss.

Third, autoregressive rollout may accumulate errors:

\[
\hat{\mathbf{q}}^{t+r\Delta t}
-
\mathbf{q}^{t+r\Delta t}
\rightarrow
\text{larger error as } r \text{ increases}.
\]

Therefore, long-horizon predictions must be validated separately from one-step predictions.

Fourth, preprocessing consistency is critical. Field ordering, normalization, graph construction, edge attributes, and target mode must remain consistent between training, fine-tuning, validation, and inference.

Finally, training cost increases with the number of nodes, edges, latent dimension, and message passing steps:

\[
\mathcal{M}
\propto
N H
+
M H
+
K M H,
\]

where \( \mathcal{M} \) denotes approximate memory demand, \( H \) is the latent dimension, and \( K \) is the number of message passing blocks.

\subsection{Summary}

Conditional MeshGraphNet was implemented as a supervised graph-based surrogate model for thermoelastic wave prediction on COMSOL finite-element meshes. The mesh is represented as

\[
\mathcal{G} = (\mathcal{V}, \mathcal{E}),
\]

where nodes store geometry, material parameters, scenario conditions, and current physical fields, while edges store relative geometric information.

The input node feature vector is

\[
\mathbf{x}_i^t
=
\begin{bmatrix}
\mathbf{p}_i \\
\mathbf{m}_i \\
\mathbf{s} \\
\mathbf{q}_i^t
\end{bmatrix}.
\]

The model learns the supervised COMSOL state-transition approximation

\[
\hat{\mathbf{Y}}^{t+\Delta t}
=
f_{\theta}
\left(
\mathbf{X}^{t},
\mathcal{G}
\right).
\]

The implemented architecture follows the Encoder--Processor--Decoder structure:

\[
f_{\theta}
=
D_{\theta_D}
\circ
P_{\theta_P}
\circ
E_{\theta_E}.
\]

The processor performs message passing by updating edge embeddings, aggregating messages with scatter-add over destination nodes, and updating node embeddings with residual connections and layer normalization. The training objective is WeightedFieldMSE over COMSOL targets, grouped into temperature, displacement, velocity, stress, strain, and other fields.

The model should therefore be interpreted as a fast learned approximation of COMSOL state transitions. It is physically motivated through mesh topology, local message passing, and material/scenario conditioning, but it does not explicitly solve the thermoelastic PDE or enforce a PDE residual during training.
