\section{Two-Dimensional Fourier Neural Operator for Thermoelastic Field Prediction}

\subsection{Model Formulation}

In the present work, the Fourier Neural Operator is used as a supervised learning model for predicting thermoelastic field evolution from simulation-generated data. The model is formulated as a two-dimensional Fourier Neural Operator, which means that the spectral transformations are applied over a two-dimensional spatial grid. The physical state at each grid point is represented by several channels, while the spatial structure of the data is described by two coordinates.

Let the computational domain be denoted by
\[
D \subset \mathbb{R}^{2}.
\]
Each point of the domain is represented as
\[
\mathbf{x} = (x,y),
\]
where \(x\) and \(y\) are the two spatial coordinates. Although the model operates on a two-dimensional grid, the predicted physical state contains multiple output channels. In the present formulation, the target field is written as
\[
\mathbf{s}(\mathbf{x}, t)
=
\left[
T(\mathbf{x}, t),
u(\mathbf{x}, t),
v(\mathbf{x}, t),
w(\mathbf{x}, t)
\right],
\]
where \(T(\mathbf{x}, t)\) is the temperature field, and \(u(\mathbf{x}, t)\), \(v(\mathbf{x}, t)\), and \(w(\mathbf{x}, t)\) are displacement-related field components extracted from the simulation data.

The purpose of the model is to learn a mapping from known input fields to future output fields. This mapping is represented as an operator
\[
\mathcal{G}_{\theta}: \mathcal{A} \rightarrow \mathcal{U},
\]
where \(\mathcal{A}\) is the input function space, \(\mathcal{U}\) is the output function space, and \(\theta\) denotes the trainable parameters of the model. The learned operator approximates the relation
\[
\mathcal{G}_{\theta}(a) \approx \mathbf{s},
\]
where \(a\) is the input field and \(\mathbf{s}\) is the target thermoelastic state.

\subsection{Temporal Prediction Task}

For a one-step temporal prediction task, the model can be written as
\[
\hat{\mathbf{s}}(\mathbf{x}, t_{n+1})
=
\mathcal{G}_{\theta}
\left(
\mathbf{s}(\mathbf{x}, t_n)
\right),
\]
where \(\mathbf{s}(\mathbf{x}, t_n)\) is the known state at time \(t_n\), and \(\hat{\mathbf{s}}(\mathbf{x}, t_{n+1})\) is the predicted state at the next time step.

If several previous time steps are used as input, the mapping becomes
\[
\hat{\mathbf{s}}(\mathbf{x}, t_{n+1})
=
\mathcal{G}_{\theta}
\left(
\mathbf{s}(\mathbf{x}, t_{n-m+1}),
\mathbf{s}(\mathbf{x}, t_{n-m+2}),
\ldots,
\mathbf{s}(\mathbf{x}, t_n)
\right),
\]
where \(m\) is the number of input time steps.

In the present implementation, the target output is represented as
\[
\hat{\mathbf{s}}(\mathbf{x}, t)
=
\left[
\hat{T}(\mathbf{x}, t),
\hat{u}(\mathbf{x}, t),
\hat{v}(\mathbf{x}, t),
\hat{w}(\mathbf{x}, t)
\right].
\]
Thus, the model predicts four physical channels at every grid point of the two-dimensional computational domain.

\subsection{Input and Output Tensor Representation}

The input tensor for a two-dimensional FNO has the form
\[
\mathbf{A}
\in
\mathbb{R}^{B \times N_x \times N_y \times C_{in}},
\]
where \(B\) is the batch size, \(N_x\) and \(N_y\) are the numbers of grid points in the two spatial directions, and \(C_{in}\) is the number of input channels. The corresponding target tensor has the form
\[
\mathbf{S}
\in
\mathbb{R}^{B \times N_x \times N_y \times C_{out}}.
\]
For the current task,
\[
C_{out}=4,
\]
because the model predicts the four fields
\[
T, \quad u, \quad v, \quad w.
\]

If the input contains \(m\) previous time steps and each time step contains \(C\) physical channels, the input tensor can be constructed by concatenating the temporal information along the channel dimension:
\[
\mathbf{A}
\in
\mathbb{R}^{B \times N_x \times N_y \times (mC)}.
\]
This representation allows the model to use information from several previous states without treating time as an additional Fourier dimension.

\subsection{Coordinate Encoding}

In many implementations of FNO, spatial coordinates are appended to the input field. For the two-dimensional case, the coordinate-augmented input is written as
\[
\tilde{a}(\mathbf{x})
=
\left[
a(\mathbf{x}),
x,
y
\right].
\]
Since
\[
\mathbf{x} = (x,y),
\]
the augmented input has dimension
\[
\tilde{a}(\mathbf{x}) \in \mathbb{R}^{d_a+2}.
\]
Coordinate encoding provides the model with explicit information about the location of each grid point. This is useful because the same physical value may have different meaning depending on its position in the domain.

\subsection{Architecture of the Two-Dimensional FNO}

The main computational block of FNO is the Fourier layer. Before applying the Fourier layers, the input is lifted to a higher-dimensional latent representation:
\[
v_0(\mathbf{x})
=
P
\left(
\tilde{a}(\mathbf{x})
\right),
\]
where \(P\) is a learnable pointwise transformation. The lifted field is
\[
v_0(\mathbf{x}) \in \mathbb{R}^{d_v},
\]
where \(d_v\) is the hidden channel dimension.

The model then applies a sequence of Fourier layers:
\[
v_0
\rightarrow
v_1
\rightarrow
\cdots
\rightarrow
v_L,
\]
where \(L\) is the number of Fourier layers. Each Fourier layer updates the hidden representation by combining a spectral transformation with a pointwise transformation:
\[
v_{l+1}(\mathbf{x})
=
\sigma
\left(
\mathcal{K}_l(v_l)(\mathbf{x})
+
W_l v_l(\mathbf{x})
+
b_l
\right),
\]
where \(l\) is the layer index, \(\mathcal{K}_l\) is the Fourier-domain operator, \(W_l\) is a pointwise linear transformation, \(b_l\) is a bias term, and \(\sigma\) is a nonlinear activation function.

After the last Fourier layer, the latent representation is projected to the target output channels:
\[
\hat{\mathbf{s}}(\mathbf{x}, t)
=
Q
\left(
v_L(\mathbf{x})
\right),
\]
where \(Q\) is a learnable projection network. Since the output contains four channels, the final prediction has the form
\[
\hat{\mathbf{s}}(\mathbf{x}, t)
\in
\mathbb{R}^{4}.
\]

The complete FNO mapping can be expressed as
\[
\mathcal{G}_{\theta}
=
Q
\circ
\mathcal{L}_{L-1}
\circ
\cdots
\circ
\mathcal{L}_{1}
\circ
\mathcal{L}_{0}
\circ
P,
\]
where \(\mathcal{L}_l\) denotes the \(l\)-th Fourier layer.

\subsection{Fourier Layer}

The spectral part of the Fourier layer is computed using the Fourier transform. For a hidden field \(v_l(\mathbf{x})\), the Fourier transform is
\[
\hat{v}_l(\boldsymbol{\xi})
=
\mathcal{F}
\left(
v_l
\right)
(\boldsymbol{\xi}),
\]
where \(\boldsymbol{\xi}\) is the frequency variable. In the two-dimensional case,
\[
\boldsymbol{\xi} = (\xi_1,\xi_2).
\]

The Fourier coefficients are multiplied by trainable complex-valued weights:
\[
\hat{z}_l(\boldsymbol{\xi})
=
R_l(\boldsymbol{\xi})
\hat{v}_l(\boldsymbol{\xi}),
\]
where \(R_l(\boldsymbol{\xi})\) is the learned spectral weight tensor.

Only a finite number of Fourier modes is retained. For the two-dimensional FNO, the retained set of modes can be written as
\[
\mathcal{M}_K
=
\left\{
(k_1,k_2)
:
|k_1| \leq K_1,
|k_2| \leq K_2
\right\},
\]
where \(K_1\) and \(K_2\) are the numbers of retained modes in the two spatial directions.

The spectral operation is therefore
\[
\hat{z}_l(k_1,k_2)
=
\begin{cases}
R_l(k_1,k_2)\hat{v}_l(k_1,k_2),
&
(k_1,k_2) \in \mathcal{M}_K,
\\
0,
&
(k_1,k_2) \notin \mathcal{M}_K.
\end{cases}
\]
After this multiplication, the inverse Fourier transform is applied:
\[
z_l(\mathbf{x})
=
\mathcal{F}^{-1}
\left(
\hat{z}_l
\right)
(\mathbf{x}).
\]

The output of the spectral operation is combined with the pointwise term:
\[
v_{l+1}(\mathbf{x})
=
\sigma
\left(
z_l(\mathbf{x})
+
W_l v_l(\mathbf{x})
+
b_l
\right).
\]
Equivalently, the Fourier layer can be written compactly as
\[
v_{l+1}
=
\sigma
\left(
\mathcal{F}^{-1}
\left(
R_l \cdot \mathcal{F}(v_l)
\right)
+
W_l v_l
+
b_l
\right).
\]

For a two-dimensional Fourier layer, the trainable spectral weights can be represented as
\[
R_l
\in
\mathbb{C}^{C_{in}^{(l)} \times C_{out}^{(l)} \times K_1 \times K_2},
\]
where \(C_{in}^{(l)}\) and \(C_{out}^{(l)}\) are the numbers of input and output hidden channels at layer \(l\). This form shows that the Fourier transformation is applied over two spatial dimensions, while the physical quantities are represented as multiple channels.

\subsection{Supervised Training Objective}

The model is trained in a purely supervised manner. This means that the training objective compares the predicted fields with the reference fields obtained from simulation data. The model does not explicitly enforce the governing partial differential equations during optimization.

Let the prediction for sample \(j\) be
\[
\hat{\mathbf{s}}_j
=
\mathcal{G}_{\theta}(a_j),
\]
and let the corresponding reference field be
\[
\mathbf{s}_j.
\]
The mean squared error loss is defined as
\[
\mathcal{L}_{MSE}
=
\frac{1}{N_s N C}
\sum_{j=1}^{N_s}
\sum_{i=1}^{N}
\sum_{c=1}^{C}
\left(
\hat{s}_{j,i,c}
-
s_{j,i,c}
\right)^2,
\]
where \(N_s\) is the number of samples, \(N\) is the number of grid points, and \(C\) is the number of output channels. In the current formulation,
\[
C=4.
\]

The relative \(L^2\) loss is defined as
\[
\mathcal{L}_{rel}
=
\frac{1}{N_s}
\sum_{j=1}^{N_s}
\frac{
\left\|
\hat{\mathbf{s}}_j
-
\mathbf{s}_j
\right\|_2
}{
\left\|
\mathbf{s}_j
\right\|_2
}.
\]

The total loss used for training can be written as
\[
\mathcal{L}
=
\lambda_{MSE}
\mathcal{L}_{MSE}
+
\lambda_{rel}
\mathcal{L}_{rel},
\]
where \(\lambda_{MSE}\) and \(\lambda_{rel}\) are weighting coefficients. If both terms are used with equal importance, then
\[
\lambda_{MSE}=1,
\qquad
\lambda_{rel}=1.
\]

No additional physics-informed residual term is included in this formulation. In particular, the loss does not contain a term of the form
\[
\mathcal{L}_{PDE}
=
\left\|
\mathcal{N}
\left(
\hat{\mathbf{s}}
\right)
\right\|_2^2,
\]
where \(\mathcal{N}\) would denote a differential operator corresponding to the governing equations. Therefore, the physical information enters the model implicitly through the simulation-generated training data, not through explicit enforcement of the partial differential equations during training.

This distinction is important for the interpretation of the model. The FNO does not solve the governing equations directly at each inference step. Instead, it learns a data-driven approximation of the input-output relation present in the simulation dataset. The learned mapping is expected to be accurate within the range of spatial patterns, material parameters, and temporal evolution represented in the training data.

\subsection{Normalization of Physical Channels}

Because the output channels may have different magnitudes, normalization is applied before training. For each channel \(c\), standard normalization can be written as
\[
\tilde{s}_{i,c}
=
\frac{
s_{i,c} - \mu_c
}{
\sigma_c
},
\]
where \(\mu_c\) is the mean value of channel \(c\), and \(\sigma_c\) is the standard deviation of channel \(c\). The inverse transformation is
\[
s_{i,c}
=
\tilde{s}_{i,c}\sigma_c + \mu_c.
\]

The normalization parameters are computed using the training set and then reused for validation, testing, and inference. This prevents information from the validation or test sets from influencing the training process.

\subsection{Evaluation Metrics}

The quality of the trained model is evaluated by comparing predicted and reference fields. The mean squared error is
\[
MSE
=
\frac{1}{NC}
\sum_{i=1}^{N}
\sum_{c=1}^{C}
\left(
\hat{s}_{i,c}
-
s_{i,c}
\right)^2.
\]
The mean absolute error is
\[
MAE
=
\frac{1}{NC}
\sum_{i=1}^{N}
\sum_{c=1}^{C}
\left|
\hat{s}_{i,c}
-
s_{i,c}
\right|.
\]
The relative error is
\[
E_{rel}
=
\frac{
\left\|
\hat{\mathbf{s}} - \mathbf{s}
\right\|_2
}{
\left\|
\mathbf{s}
\right\|_2
}.
\]

For channel-wise evaluation, the relative error of channel \(c\) can be written as
\[
E_c
=
\frac{
\left\|
\hat{s}_{c} - s_{c}
\right\|_2
}{
\left\|
s_{c}
\right\|_2
}.
\]
This metric is useful because the model predicts several physical fields, and each field may have a different scale and different prediction difficulty.

For time-dependent prediction, the error can also be evaluated as a function of the prediction step:
\[
E(r)
=
\frac{
\left\|
\hat{\mathbf{s}}(\cdot, t_{n+r})
-
\mathbf{s}(\cdot, t_{n+r})
\right\|_2
}{
\left\|
\mathbf{s}(\cdot, t_{n+r})
\right\|_2
},
\]
where \(r\) is the forecasting step. This expression is especially useful when the model is applied recursively for several future time steps.

\subsection{Hyperparameters}

The most important hyperparameters of the two-dimensional FNO are the number of Fourier layers \(L\), the number of retained modes \(K_1\) and \(K_2\), the hidden channel dimension \(d_v\), the number of input time steps \(m\), the learning rate \(\eta\), and the batch size \(B\). These parameters control the model capacity, computational cost, and stability of training.

The number of retained modes determines how much spectral information is used by the model. If the number of modes is too small, the model may lose important spatial details. If the number of modes is too large, the model becomes more computationally expensive and may require more training data. The hidden channel dimension controls the amount of information stored in the latent representation, while the number of Fourier layers controls the depth of the operator approximation.

\subsection{Summary}

In summary, the FNO used in this project is a two-dimensional field-to-field prediction model. It applies Fourier transformations over the two spatial dimensions \(x\) and \(y\), while the physical state is represented through multiple channels. The model predicts four target fields,
\[
T, \quad u, \quad v, \quad w,
\]
and is trained using supervised loss functions based on mean squared error and relative \(L^2\) error. No explicit PDE residual is included in the training objective. Thus, the model learns the thermoelastic behavior from simulation data rather than by directly enforcing the governing equations during optimization.
